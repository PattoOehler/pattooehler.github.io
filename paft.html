PAFT

Peer to Peer Anonymous File Transfer

  • Code


DHT - Distributed Hash Table

 This program uses a DHT to be able to find peers and files in log(n) time with high probability. This paper was used as a reference while building the DHT from scratch. A notable difference is that PAFT uses TCP while the paper describes using UDP, this creates less communication problems that need to be solved at the expense of speed.

Lifecycle of a File

Connecting to the Network

 First to do anything you have to have someone to connect to, you can find peers by having them already stored in the DHT or by pinging someone who is already in the network.

Uploading a file

 To upload a file you first have to make a metadata file. This separates the file into chunks and contains information such as how many chunks are in the file, the checksum of those chunks, the file ID and filename. After being created a string is given that contains the checksum and the ID of the file for others to find the file and download it. Once the metadata file is made then you find the closest 3 peers in the network to that file ID and ask them to store the File ID and your IP/Port.

Downloading a file

 To download a file all you need is the ID of the file and the checksum of the metadata file. First find the closest peers in the network to the file ID asking them if they know who has the metadata file. Download the metadata file from who was found, comparing the checksum. Continue to download all of the chunks and validate each one. Once you have the file, upload the file to the network so that others can download the file from you as well.

Downloading a file using Onion Routing(Beta)

 The only difference between using onion routing and a normal download is that you run your request through multiple peers so that the uploader does not know who is downloading the file. Those peers forward the request to the next peer and give the response back. This gives the downloader anonymity but requires 3x the data to be transfered.
